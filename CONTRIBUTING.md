# Contributing to Awesome Claude AI

Thank you for your interest in contributing! This document provides guidelines for contributing to this awesome list of Claude AI tools, resources, and applications.

## How to Contribute

### Adding a New Resource

1. **Fork the repository**
2. **Create a new branch** for your addition
3. **Add your resource** in the appropriate section
4. **Ensure quality** - only add resources that are:
   - Claude-specific or have Claude integration
   - High quality and well-maintained
   - Accessible (preferably free or with free tier)
   - Well-documented
   - Not already listed
5. **Follow the format** described below
6. **Submit a pull request**

### Resource Format

Please use the following format for new entries:

```markdown
- [Tool Name](https://url-to-tool.com) - Brief description of what it does
  - Key feature 1
  - Key feature 2
  - Relevant details (Claude models supported, pricing, etc.)
```

**Guidelines:**
- Keep descriptions concise (1-2 sentences)
- Start with a capital letter
- End with a period
- Be objective and factual
- Avoid marketing language
- Mention Claude-specific features

### Categories

Place your resource in the most appropriate category:

- **Official Resources** - Anthropic's official tools and documentation
- **Models & Capabilities** - Information about Claude models
- **API & SDKs** - Development kits and API clients
- **Developer Tools** - CLI tools, IDE integrations, browser extensions
- **Prompt Engineering** - Prompting guides and techniques
- **Applications & Use Cases** - Specific application categories
- **Integrations** - Platforms, frameworks, workflows
- **Community Projects** - Open source tools and experiments
- **Learning Resources** - Guides, courses, papers
- **Comparisons & Benchmarks** - Performance and model comparisons
- **Best Practices** - Guidelines and recommendations

If none fit, propose a new category in your pull request.

## Pull Request Process

1. **Check existing PRs** to avoid duplicates
2. **One resource per PR** (makes review easier)
3. **Clear PR title**:
   - "Add [Tool Name]"
   - "Update [Section Name]"
   - "Fix broken link in [Section]"
4. **Describe your addition**:
   - What you're adding
   - Why it's valuable
   - How it relates to Claude
   - Any affiliation disclosure
5. **Wait for review** - maintainers will review within a few days
6. **Address feedback** if requested
7. **Celebrate** when merged! üéâ

## Disclosure Requirements

**If you have any affiliation with the tool you're adding (creator, employee, contributor, etc.), you MUST disclose this in your PR.**

Example:
```
Adding [Tool Name] - I'm a developer on this project
```

## Style Guide

### Tool Descriptions

**Good examples:**
```markdown
- [Claude Code](https://claude.ai/claude-code) - Official terminal-based coding assistant by Anthropic
  - File operations and editing
  - Bash command execution
  - MCP integration
  - Multi-file context

- [Aider](https://aider.chat/) - AI pair programming in the terminal with Claude support
  - Git integration
  - Multi-file editing
  - Claude 3.5 Sonnet optimized
  - Free and open source
```

**Bad examples:**
```markdown
- [Amazing AI Tool](https://example.com) - The best AI tool ever!!!
- [My Startup](https://startup.com) - Revolutionary Claude integration
- [Tool](https://example.com) - Uses Claude
```

### Markdown Style

- Use proper Markdown syntax
- Keep formatting consistent
- Use bullet points for features
- Bold for emphasis (sparingly)
- No excessive emoji

### Links

- Use HTTPS when available
- Link to official sites
- Verify links work before submitting
- Use full URLs (not shortened links)

## Claude-Specific Guidelines

When adding Claude-related resources:

### Model Support
- Specify which Claude models are supported
- Mention if it uses latest models (3.5 Sonnet, 3.5 Haiku)
- Note any model-specific features

### API Integration
- Indicate if it uses official Anthropic API
- Mention if it supports streaming
- Note tool use/function calling support
- Specify vision capabilities if relevant

### Pricing/Access
- Note if it's free, freemium, or paid
- Mention API key requirements
- Indicate self-hosted options if available

### Documentation
- Link to Claude-specific documentation
- Include setup guides if available
- Reference example code if exists

## Comparison & Benchmark Guidelines

When adding comparisons or benchmarks:

1. **Be objective** - Use verifiable data
2. **Cite sources** - Link to benchmark sources
3. **Keep updated** - Note the date and Claude version
4. **Be comprehensive** - Include context
5. **Explain methodology** - How was it measured?

## Best Practices for Guides

When adding guides or best practices:

1. **Be practical** - Include actionable advice
2. **Provide examples** - Show, don't just tell
3. **Link to sources** - Give credit to Anthropic or others
4. **Stay current** - Claude evolves quickly, note versions
5. **Be inclusive** - Consider different use cases
